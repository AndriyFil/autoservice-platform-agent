# AutoService Learning Notes

This folder is for concise learning notes about Laravel, PHP, and backend concepts we encounter while implementing AutoService.

Learning notes are for understanding how a concept works in this project. They are not production architecture decisions, product documentation, or a replacement for Laravel documentation.

Create a note when a concept appears during implementation or when it was confusing enough that a project-specific example would help.

Good candidates include:

- Route Model Binding
- Service Container
- Dependency Injection
- Form Requests
- Middleware
- Policies
- Eloquent Relationships
- Eager Loading
- Transactions
- Jobs / Queues
- Events / Listeners
- Testing

Rules for learning notes:

- Do not copy Laravel documentation.
- Explain using examples from the current AutoService codebase.
- Keep notes concise and practical.
- Prefer real files, classes, and code snippets from this project.
- Do not invent features.
- If AutoService does not yet use the concept, mark the project usage section as `TODO`.
- Use plain language.
- Include a SQL equivalent when it makes the mechanism easier to understand.
- Keep the note focused on learning, not on choosing or defending architecture.
